# OpenTranscribe RU

Бесплатная, быстрая транскрибация аудио и видео на русском языке прямо в браузере с использованием моделей Whisper (через Transformers.js).

## Как запустить локально

### Требования

*   [Node.js](https://nodejs.org/) (версия 18 или выше)
*   npm (обычно устанавливается вместе с Node.js)

### Установка и запуск

1.  **Распакуйте архив** с исходным кодом в папку.
2.  **Откройте терминал** (командную строку) в этой папке.
3.  **Установите зависимости**, выполнив команду:
    ```bash
    npm install
    ```
4.  **Запустите проект** в режиме разработки:
    ```bash
    npm run dev
    ```
5.  Откройте ссылку, которая появится в терминале (обычно `http://localhost:5173`), в вашем браузере.


## Запуск на Windows

### Быстрый запуск (BAT-файл)

1. Убедитесь, что установлен **Node.js 18+** (вместе с npm).
2. В проводнике откройте папку проекта и дважды кликните `start-windows.bat`.
3. Скрипт автоматически:
   - проверит `node` и `npm`;
   - выполнит `npm install`;
   - запустит dev-сервер на `http://localhost:5173`.

### Ручной запуск (PowerShell)

```powershell
cd C:\path\to\openTranslate
npm install
npm run dev
```

### Ручной запуск (CMD)

```cmd
cd C:\path\to\openTranslate
npm install
npm run dev
```


## Режимы модели и ускорения

- В интерфейсе теперь доступны модели: **Tiny**, **Base**, **Large v3 Turbo**.
- Для обработки можно выбрать:
  - **CPU (WASM)** — максимально совместимо;
  - **GPU (WebGPU)** — быстрее, но нужна поддержка WebGPU браузером/драйвером.
- Если на GPU зависает или падает загрузка, переключитесь на **CPU (WASM)** и/или на модель поменьше.


## Как опубликовать последнюю версию (Codex + GitHub)

### В Codex

1. Убедитесь, что изменения внесены и проверены (`npm run build`).
2. Попросите Codex сделать commit в текущей ветке.
3. Попросите Codex создать PR (pull request) с понятным названием и описанием.
4. Проверьте, что в PR есть все нужные файлы и нет лишних.

### На GitHub

1. Откройте созданный PR в репозитории.
2. Пройдите проверки (если настроены CI/lint/build).
3. Нажмите **Merge pull request** (обычно `Squash and merge`).
4. После merge обновите локальную `main`:
   ```bash
   git checkout main
   git pull
   ```
5. (Опционально) Если нужен деплой, запустите ваш обычный workflow публикации (например GitHub Pages / Vercel / сервер).

> Если хотите, можно повторять этот цикл для каждой правки: **изменение → commit → PR → merge → деплой**.


## Если висит на `Загрузка: config.json` или `tokenizer_config.json`

Это частая ситуация при первом скачивании модели (особенно Large) или при проблемах сети.

Что делать:
1. Подождите 1–2 минуты (первая загрузка может быть долгой).
2. Переключите устройство на **CPU (WASM)**.
3. Выберите модель поменьше (**Tiny** или **Base**) и попробуйте снова.
4. Отключите VPN/прокси/корпоративный фильтр и перезапустите страницу.
5. Откройте DevTools → Console/Network и проверьте, нет ли блокировок загрузки с Hugging Face.


### Важно про загрузку моделей

- Модели Whisper **не лежат в репозитории**, они скачиваются при первом запуске из Hugging Face.
- Нужен стабильный интернет без блокировок CDN/Hugging Face.
- Если GPU-режим не стартует, приложение теперь автоматически пробует CPU (WASM).

- Перед загрузкой pipeline выполняется pre-check доступности ключевых файлов модели (`config.json`, `tokenizer_config.json`).
- При ошибках теперь показывается явный технический текст ошибки в интерфейсе.
- Если `decodeAudioData` не поддерживает конкретный контейнер/кодек, используется fallback через `<audio>` + `MediaElementAudioSource`.
